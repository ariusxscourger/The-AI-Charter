Workflow

How It Works

  1. Team submits AI feature proposal via web form
  2. Orchestrator creates Band.ai room, posts submission context
  3. 5 agents fire concurrently , each runs LLM-powered domain evaluation
  4. Cross-examination: agents read peer findings, challenge if needed
  5. Each agent casts a vote (approve / reject / flag)
  6. Record generator compiles full transcript into governance record
Message Types in Band Room
  • submission_context , Full proposal payload
  • status_update , Agent progress
  • findings , Structured domain results
  • challenge , Cross-examination debates
  • vote , Final verdict with reasoning
Band.ai Deep Dive

Band = Collaboration Layer

  • Band room IS the audit trail , no separate status DB needed
  • Agents discover each other, join rooms, and exchange structured context
  • Cross-examination happens inside Band: agents challenge peer findings in real-time
  • sessionId = room_id , clean, traceable architecture
  • Full transcript compiles into defensible governance record
Why Band Matters

Differentiation

Key Differentiators

Deterministic Voting
LLMs power evaluation and reasoning, but the vote itself is pure Python logic. Reproducible, auditable, not subject to hallucination at the decision layer.
Concurrent 5-Agent Execution
All agents run in parallel via asyncio. Security alone runs 7 sub-domain evaluations concurrently with rate limiting and retry logic.
Resilient Error Handling
Band SDK falls back to local mock rooms. LLM client retries 10x with exponential backoff. Agents degrade gracefully on failure.
Provider-Flexible LLM
Supports 3 LLM providers (Featherless, AI/ML API, OpenRouter) with automatic fallback. Open-source models by default, near-zero cost.
